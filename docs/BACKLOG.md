The user reported a regression: "I am actively working on the app and I got the message that my session has expired. How can that happen. it should happen only on inactivity." The audit traced the root cause to a three-part architectural flaw — `infra/cognito.tf` never overrode Cognito's default `id_token_validity` (60 min), `frontend/lib/auth/cognito.ts:signIn()` captured only the `IdToken` from the Cognito response and silently discarded the `RefreshToken`, and `SessionExpiryWatcher` scheduled a hard `setTimeout(() => markSessionExpired("expired"), exp)` at the JWT's `exp` claim regardless of activity.
