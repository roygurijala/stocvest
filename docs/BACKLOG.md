| B33 | **Per-symbol funnel transparency / near-miss panel on Scanner** | **Intentionally deferred from the empty-state polish that just shipped.** When the Scanner is empty, today's `ScannerEmptyStateCard` answers _aggregate_ "why nothing fired" (regime, session state, universe scanned, mode-specific re-enable bullets). The next-level version would answer _per-symbol_: "we evaluated NVDA today; it failed at the volume / momentum gate by N% RVOL," or "AAPL passed regime but failed the score floor at 0.42 vs the 0.48 minimum." **Why this is not P1/P2:** (a) **implicit-recommendation risk** — a per-symbol "almost made it" panel is the same UX class as "near-miss," which the user could reasonably read as "watch this one closely." The legal / framing copy would have to be exceptional ("symbols our engine evaluated but did not qualify — this list is not a watchlist"). Shipping this requires a new structured trace from `DailyBarScanner` + the day-side scanner, plus a `GET /v1/signals/scanner-trace?mode=swing\|day` endpoint with bounded retention (probably 24-48h rolling, in DynamoDB or S3 Parquet). **Scope when promoted:** (1) backend trace emission + endpoint + IAM. (2) Frontend: optional "Show today's evaluations" expander under `ScannerEmptyStateCard`, capped to 20 rows, with explicit "Evaluated · did not qualify · NOT a watchlist" copy.
